## geometry: the ontology of georeferenced stuff 
    - what is this object?
    - where is it?
    - what area does it occupy?
    - what does it belong to?
    - how can it be serialized as metadata?

    - the things that exist in space 
        - mosaics -> source raster metadata 
            - id 
            - uri 
            - crs 
            - bounds 
            - width/height 
            - bands 
            - dtype 
            - nodata 
        - tiles -> windowed subregion of mosaic
            - tile id
            - parent id
            - source uri 
            - window bounds 
            - x/y offset 
            - width/height 
            - trandform 
            - shard reference if exists 
            - label if exists 
        - paths -> vector feature of mosaic 
            - path id
            - parent id
            - line feature 
            - polygon feature 
            - annotations 
            -   confidence 
            -    optional level of damage 
        ( metamosaics, which aggregate these for a region in spacce)
            - metamosaic id 
            - canonical crs 
            - overall bounds 
            - member ids / dimension 
    - mostly stores metadata-first:
        - source uri 
        - stable id 
        - CRS 
        - Footprint 
        - shape 
        - window offsets?
        - parent/child relations 
        - reference to storage 
    - does not store 
        - full raster arrays 
        - shard payloads 
        - decoded geometry collections 
        - low level logic 
    - Tile Mosaic Path should contain enough metadata to let
            another layer retrieve the pixels safely 
    
# repositories: typed collection access for geometry 
    - how do i retrieve a tile?
    - how do i stream tiles for this mosaic?
    - how do i find all paths intersecting a point?
    
    - exposes domain questions about geometry:
        - get by id 
        - list by parent 
        - query by point/bounds 
        - persist/update resource metadata 
    - MosaicRepo
    - TileRepo 
        - get(tile_id)
        - by_mosaic(mosaic_id)
        - at_point(x,y,crs)
        - neighbors(tile_id)
        - put_many(TILES)
    - PathRepo 
    - MetaMosaicRepo 

# operators: geospatial transformations of geometries 
    - how do i turn a mosaic into a lower resolution?
    - how do i turn a mosaic into a set of tile?
    - how do i assign labels to tiles given a path?
    - how do i pack tiles into shards?
    - how do i compute neoghbor relations?

    
    - DownsampleOperator(MOSAIC, DSPARAMS) -> MOSAIC 
    - TesselateOperator(MOSAIC, TPARAMS) -> ITerator[TILES]
    - QuantizeOperator(QUANTIZE, QPARAMS) -> TILE 
    - LabelOperator(Iterator[TILES], paths, LabelParams) -> Iterator[LABELS]
    - ShardOperator(tiles: Iterator[TILES], ShardSpec) -> TILESHARDS
    - RelateOperator 

# SPECS: Typed params of operators 
    - what tile size? Stride 
    - what output dtype?
    - what resampling method? 
    - what shard size for outputs?
